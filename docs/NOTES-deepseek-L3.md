## Design choices

### 1. DelimitedExtractor parse rule

`extract(text)` is a pure, deterministic line parser (`src/semantic/fact.ts`):

1. Split `text` on `\n`.
2. For each line, split on `this.delimiter` (default `"\t"`).
3. `trim()` every field.
4. Keep the line as a `Triple {subject, predicate, object}` **iff** it split into EXACTLY three
   fields AND none of the three is empty after trim. Otherwise skip the line (no throw).
5. Preserve line order in the returned array.

This skips: blank lines, lines with the wrong field count (e.g. a 4-field line), and lines with any
empty field. It is non-semantic by design — it exists so the graph mechanics are byte-reproducible
and golden-pinnable, the L3 analogue of L0's fixed ciphertext and L2's `HashEmbedder`.

### 2. Fact identity / dedup + canonical sort key

`LocalKnowledgeGraph` stores facts in a `Map<string, Fact>` keyed by the compound identity
`subject \x00 predicate \x00 object \x00 sourceObjectId` (NUL separator, which cannot occur in the
delimited corpus). Consequences:

- **Dedup:** re-`addFact` of an identical fact overwrites the same key → no-op; `size()` is the map
  size (distinct fact count).
- **Provenance is part of identity:** the SAME triple from two different `sourceObjectId`s yields two
  distinct keys → two facts. `removeBySource(id)` iterates and deletes every entry whose
  `sourceObjectId === id`.
- **Canonical order** (applied to every `match`/`neighbors` result): ascending by
  `(subject, predicate, object, sourceObjectId)` via plain string `<`/`>` comparison. `entities()`
  returns the distinct union of subjects and objects, sorted ascending. Determinism here is what
  makes the semantic golden reproducible with EXACT string comparison (no tolerance needed — all
  values are strings).
- `match` treats an omitted pattern field as a wildcard; the empty pattern returns all facts.

### 3. Semantic facade

`createSemantic({extractor, graph})` (`src/semantic/semantic.ts`) wires the seams with no extra
state. `ingestObject(objectId, source)` resolves `{text}` via `extractor.extract` or takes `{triples}`
directly, adds each as `{triple, sourceObjectId: objectId}`, and returns the count of triples
ingested. `rebuild(stream)` does `graph.clear()` then re-extracts+adds over the caller's async
`{objectId, text}` stream, returning `graph.size()` afterwards. `query`/`neighbors`/`removeObject`
pass straight through to the graph. **No decryption, no KEK, no `crypto` import** (the D6 boundary).

### 4. Out-of-root invariant test construction

- **Behavioural** (`test/semantic-l3.test.ts`): runs the L0 spine scenario once, recording the final
  `vault_memory_root`, the first `object_id`, and the first `space_state`; runs the SAME scenario on
  a fresh spine while ALSO `ingestObject`-ing every appended object and issuing `query`/`neighbors`
  calls; then asserts `rootWith === rootWithout`, `object_id` equal, and `space_state` equal. Proves
  L3 activity touches no spine commitment.
- **Structural**: reads every `.ts` under `src/spine/` and `src/canonical/` and fails on any
  `semantic` import — enforcing the one-way dependency (semantic → spine types only).

### 5. Semantic golden placement and what it pins

`vectors/semantic/golden.json` (fourth sibling to spine/anchor/recall), `_status` PRE-NORMATIVE,
generated by `scripts/semantic-scenario.ts` (wired into `scripts/generate-vectors.ts`). The fixed
scenario uses the default tab `DelimitedExtractor` over a 5-object corpus that deliberately exercises
the skip paths (a 4-field line and a blank line in `obj:delta`), duplicate cross-source triples, and
shared entities. It pins: the extracted triples per object, the full `entities()` list, the
`fact_count`, six `match` queries (subject-only, predicate-only, object-only, full, empty/all,
no-match) and four `neighbors` queries (incl. an unknown entity). The golden test asserts an EXACT
`deepEqual` against the committed scenario plus that every `match` result is canonically sorted.
