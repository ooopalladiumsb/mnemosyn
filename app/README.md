# Mnemosyne D14 — Telegram Mini App

A sovereign AI memory agent available as a Telegram Mini App. Static frontend on
GitHub Pages, dynamic backend on a separate HTTPS host using `bun`.

**Bot:** @MnemoVaultBot (registered via @BotFather)

## Architecture

```
Telegram Mini App (app/index.html)
  └─ GitHub Pages (https://<user>.github.io/mnemosyn/)
       │
       │ POST /turn  (x-telegram-init-data header + CORS)
       ▼
   backend server (app/server.ts)
     └─ Bun HTTP server
          ├─ TelegramInitDataAuthenticator → per-user vault
          ├─ withCors → allows Pages origin
          └─ createAgentHandler → deepseekBrain + embedder + FileSpineStore
```

## Deploy the frontend (GitHub Pages)

1. Edit `app/index.html` — set `BACKEND_URL` to your backend's public HTTPS URL.
2. Publish the `app/` directory to GitHub Pages:
   - Push to a repo (e.g. `ooopalladiumsb.github.io` repo, in a `/mnemosyn` folder)
   - Or use a dedicated repo with GitHub Pages enabled on the `main` branch
3. The frontend is entirely static — no build step, no npm.

## Deploy the backend

1. Install [bun](https://bun.sh) and the deps:
   ```bash
   cd projects/mnemosyne
   bun install
   ```

2. Set the required environment variables:
   ```bash
   export BOT_TOKEN="123456:ABC-DEF..."  # from @BotFather
   export VAULT_SECRET="<64 hex chars>"  # generate: openssl rand -hex 32
   export DEEPSEEK_API_KEY="sk-..."      # from DeepSeek API
   export EMBED_API_KEY="jina_..."       # from Jina AI (or Qwen/Gemini)
   export EMBED_BASE_URL="https://api.jina.ai/v1"  # optional, default Jina
   export EMBED_MODEL="jina-embeddings-v3"         # optional
   export EMBED_DIM="1024"                          # optional, default 1024
   export ALLOWED_ORIGIN="https://ooopalladiumsb.github.io"  # your Pages origin
   export DATA_DIR="./data"     # persistent storage for FileSpineStore
   export PORT="3000"
   ```

3. Expose the backend over HTTPS (required for Telegram Mini Apps):
   - Option A: Use [cloudflared](https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/):
     ```bash
     cloudflared tunnel --url http://localhost:3000
     ```
   - Option B: Use a VPS with a reverse proxy (nginx + Let's Encrypt).

4. Start the backend:
   ```bash
   bun run app/server.ts
   ```

5. Update `BACKEND_URL` in `app/index.html` to your public HTTPS URL (e.g.
   `https://mnemosyne-backend.example.com`).

## Local testing

For local development, you can test the backend without a real Telegram environment:

```bash
# Start backend
bun run app/server.ts

# Test health endpoint
curl http://localhost:3000/health

# Test turn (using HeaderAuthenticator for dev)
curl -X POST http://localhost:3000/turn \
  -H "Content-Type: application/json" \
  -H "x-vault-did: memory://vault/<your-test-did>" \
  -d '{"input":"hello"}'
```

To test with real Telegram initData, you must open the Mini App inside Telegram. The
backend's `TelegramInitDataAuthenticator` validates the `initData` string using the bot
token HMAC.

## Secrets

All secrets live server-side only:
- `BOT_TOKEN` / `VAULT_SECRET` — used only in `TelegramInitDataAuthenticator`
- `DEEPSEEK_API_KEY` / `EMBED_API_KEY` — used only in the per-vault agent factory
- Never appear in responses, frontend code, or logs
- The frontend only sends the pre-signed `initData` (generated by Telegram) — the bot
  token never leaves the backend
